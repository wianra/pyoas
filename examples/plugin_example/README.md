# pyoas Plugin Example — Copyright Header Injection

This example shows how to write a pyoas plugin that post-processes every
generated Pydantic model file to prepend a custom copyright header.

## How it works

`myapp_plugin.py` implements the `Plugin` protocol from `pyoas.core.plugins`:

| Hook | What this plugin does |
|---|---|
| `on_spec_loaded` | No-op — returns spec unchanged |
| `on_model_file_written` | Prepends `HEADER` to every model file |
| `on_router_file_written` | No-op — returns content unchanged |
| `on_generate_complete` | Prints a summary line to stdout |

## Activation

### Option A — explicit entry in `pyoas.yaml`

```yaml
plugins:
  - myapp_plugin:HeaderPlugin
```

`myapp_plugin.py` must be importable (i.e. on `sys.path` or installed as a
package). If it lives next to your `pyoas.yaml`, run pyoas from that directory.

### Option B — `pyproject.toml` entry-point (recommended for distributable plugins)

```toml
[project.entry-points."pyoas.plugins"]
header = "myapp_plugin:HeaderPlugin"
```

After installing the package (`pip install -e .`), pyoas discovers the plugin
automatically — no `plugins:` entry in `pyoas.yaml` needed.

## Testing the plugin

```shell
# From this directory
uv run pyoas init openapi.yaml   # or point at your own spec

# Add to pyoas.yaml:
#   plugins:
#     - myapp_plugin:HeaderPlugin

uv run pyoas models              # model files will begin with the header
head -5 src/generated/models/pets.py
```

Expected output in the generated file:

```python
# ----------------------------------------------------------------
# Auto-generated by pyoas. DO NOT EDIT.
# Copyright (c) 2026 My Company. All rights reserved.
# ----------------------------------------------------------------
from __future__ import annotations
...
```

## Verifying with doctor

```shell
uv run pyoas doctor
```

If the plugin cannot be imported, `doctor` reports a `plugin_load` error
before any generation takes place.

## Using this as a template

1. Copy `myapp_plugin.py` to your project.
2. Rename `HeaderPlugin` and `name`/`version` attributes.
3. Implement the hooks you need; return `content` unchanged in the others.
4. Add the plugin to `pyoas.yaml` or register it as an entry-point.
